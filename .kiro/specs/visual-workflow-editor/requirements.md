# 可视化工作流编辑器 - 需求文档

## 1. 项目概述

将AI生成的Playwright脚本转换为可视化的工作流编辑器，类似Automa的block-based设计。用户可以通过拖拽、连接功能块来创建和修改爬虫工作流，无需编写代码。

## 2. 用户故事

### 2.1 作为普通用户
- 我希望看到AI生成的脚本以可视化的方式展示，而不是代码
- 我希望能够通过拖拽添加、删除、修改功能块
- 我希望能够连接不同的功能块来定义执行顺序
- 我希望能够保存和执行可视化工作流

### 2.2 作为高级用户
- 我希望能够在可视化编辑器和代码编辑器之间切换
- 我希望能够导入/导出工作流配置
- 我希望能够复用常用的工作流模板

## 3. 功能需求

### 3.1 核心Block类型

#### 3.1.1 浏览器控制类
- **访问页面** (Navigate)
  - 输入：URL
  - 配置：等待策略、超时时间
  
- **滚动页面** (Scroll)
  - 配置：滚动方式（智能/固定次数）、等待时间
  
- **等待** (Wait)
  - 配置：等待时间（毫秒）

#### 3.1.2 页面交互类
- **点击元素** (Click)
  - 输入：选择器
  - 配置：等待元素出现
  
- **输入文本** (Type)
  - 输入：选择器、文本内容
  
- **选择下拉框** (Select)
  - 输入：选择器、选项值

#### 3.1.3 数据提取类
- **提取元素** (Extract)
  - 输入：选择器
  - 配置：提取属性（text/src/href等）
  - 输出：数据数组
  
- **提取图片** (Extract Images)
  - 配置：过滤条件
  - 输出：图片列表
  
- **提取链接** (Extract Links)
  - 配置：过滤条件
  - 输出：链接列表

#### 3.1.4 逻辑控制类
- **条件判断** (Condition)
  - 输入：条件表达式
  - 输出：true/false分支
  
- **循环** (Loop)
  - 输入：循环次数或数据源
  
- **日志输出** (Log)
  - 输入：消息内容

#### 3.1.5 数据处理类
- **数据转换** (Transform)
  - 输入：数据
  - 配置：转换规则
  
- **数据过滤** (Filter)
  - 输入：数据数组
  - 配置：过滤条件

### 3.2 可视化编辑器

#### 3.2.1 画布功能
- 拖拽添加block
- 连线连接blocks
- 缩放和平移画布
- 网格对齐

#### 3.2.2 Block操作
- 点击编辑block配置
- 删除block
- 复制/粘贴block
- 拖动调整位置

#### 3.2.3 连接管理
- 拖拽创建连接
- 删除连接
- 显示数据流向

### 3.3 脚本转换

#### 3.3.1 代码解析
- 解析Playwright脚本
- 识别常见模式（goto、click、evaluate等）
- 转换为block配置

#### 3.3.2 代码生成
- 将blocks转换为Playwright代码
- 保持统一的数据格式
- 生成可执行的脚本

### 3.4 工作流执行
- 验证工作流完整性
- 按顺序执行blocks
- 实时显示执行状态
- 错误处理和回滚

## 4. 验收标准

### 4.1 可视化编辑器
- [ ] 能够拖拽添加至少10种基础block类型
- [ ] 能够通过连线定义block执行顺序
- [ ] 能够编辑block的配置参数
- [ ] 能够删除block和连接
- [ ] 画布支持缩放和平移

### 4.2 脚本转换
- [ ] 能够解析AI生成的Playwright脚本
- [ ] 能够将脚本转换为可视化blocks
- [ ] 能够将blocks转换回可执行的Playwright代码
- [ ] 转换前后功能保持一致

### 4.3 工作流执行
- [ ] 能够执行可视化工作流
- [ ] 能够实时显示执行进度
- [ ] 能够处理执行错误
- [ ] 执行结果符合统一数据格式

### 4.4 用户体验
- [ ] 界面直观易用
- [ ] 操作响应流畅
- [ ] 提供操作提示和帮助
- [ ] 支持撤销/重做操作

## 5. 技术约束

- 前端使用Vue3 + TypeScript
- 使用现有的Vue3项目结构
- 可视化编辑器可以使用Vue Flow或类似库
- 后端继续使用Express + Playwright
- 保持与现有爬虫agent的兼容性

## 6. 非功能需求

- 性能：编辑器操作响应时间 < 100ms
- 可用性：新用户5分钟内能创建简单工作流
- 兼容性：支持Chrome、Edge等现代浏览器
- 可维护性：代码结构清晰，易于扩展新的block类型
