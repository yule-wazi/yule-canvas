# 循环变量功能使用指南

## 功能概述

循环模块现在支持灵活的循环变量配置，包括：
- 可选的循环变量（可以启用或禁用）
- 自定义循环变量名称
- 三种起始值模式：默认、自定义数值、全局变量

## 使用步骤

### 1. 启用循环变量

在循环模块的属性面板中：
1. 找到"使用循环变量"选项
2. 点击切换按钮启用（按钮会变成蓝色显示"已启用"）
3. 启用后会显示更多配置选项

### 2. 配置循环变量名

在"循环变量名"输入框中输入变量名，例如：
- `index` - 常用的索引变量
- `i` - 简短的循环计数器
- `num` - 数字变量
- 任何有效的 JavaScript 变量名

### 3. 选择起始值类型

有三种起始值模式可选：

#### 默认（从1开始）
- 循环变量从 1 开始递增
- 适合大多数场景，如 `nth-child(1)`, `nth-child(2)`, ...

#### 自定义数值
- 可以指定任意起始数值
- 例如：设置为 5，则循环变量从 5 开始：5, 6, 7, ...
- 适合需要特定起始索引的场景

#### 使用全局变量
- 从下拉框中选择已定义的全局变量
- 循环变量将从该全局变量的值开始
- 适合动态配置起始值的场景

## 使用示例

### 示例 1：基本用法（从1开始）

**配置：**
- 使用循环变量：✅ 已启用
- 循环变量名：`index`
- 起始值类型：默认（从1开始）
- 循环次数：5

**在选择器中使用：**
```
div:nth-child({{index}})
```

**实际效果：**
- 第1次循环：`div:nth-child(1)`
- 第2次循环：`div:nth-child(2)`
- ...
- 第5次循环：`div:nth-child(5)`

### 示例 2：自定义起始值

**配置：**
- 使用循环变量：✅ 已启用
- 循环变量名：`num`
- 起始值类型：自定义数值
- 起始值：3
- 循环次数：5

**在选择器中使用：**
```
.item-{{num}}
```

**实际效果：**
- 第1次循环：`.item-3`
- 第2次循环：`.item-4`
- ...
- 第5次循环：`.item-7`

### 示例 3：使用全局变量作为起始值

**前提：**
先在"🔢 变量"管理中定义全局变量：
- 变量名：`startIndex`
- 值：`10`

**配置：**
- 使用循环变量：✅ 已启用
- 循环变量名：`pageNum`
- 起始值类型：使用全局变量
- 选择变量：`startIndex`
- 循环次数：3

**在选择器中使用：**
```
a[data-page="{{pageNum}}"]
```

**实际效果：**
- 第1次循环：`a[data-page="10"]`
- 第2次循环：`a[data-page="11"]`
- 第3次循环：`a[data-page="12"]`

### 示例 4：不使用循环变量

**配置：**
- 使用循环变量：❌ 已禁用
- 循环次数：5

**说明：**
- 不会创建循环变量
- 循环体内的模块不能使用 `{{变量名}}` 语法
- 适合只需要重复执行固定操作的场景

## 注意事项

1. **变量名规则**：
   - 必须是有效的 JavaScript 变量名
   - 不能与全局变量重名
   - 建议使用有意义的名称

2. **起始值**：
   - 默认模式：从 1 开始
   - 自定义模式：可以是任意整数（包括负数）
   - 全局变量模式：会自动转换为数字

3. **变量作用域**：
   - 循环变量只在循环体内有效
   - 每次循环迭代都会更新变量值
   - 变量值 = 循环索引（从0开始）+ 起始值

4. **与全局变量的区别**：
   - 循环变量：每次循环自动递增
   - 全局变量：固定值，不会自动变化
   - 可以组合使用：用全局变量设置循环变量的起始值

## 技术实现

### 代码生成逻辑

当启用循环变量时，生成的代码类似：

```javascript
for (let __loopIndex = 0; __loopIndex < count; __loopIndex++) {
  const variableName = __loopIndex + startValue;
  // 循环体代码...
}
```

其中：
- `__loopIndex`：内部循环计数器（从0开始）
- `variableName`：用户定义的循环变量名
- `startValue`：起始值（1、自定义数值或全局变量的值）

### 变量替换

在循环体内的模块中：
- `{{variableName}}` 会被替换为 `${variableName}`
- 使用模板字符串语法，在运行时求值
- 日志输出会显示实际的变量值

## 常见问题

**Q: 为什么我的循环变量总是从1开始？**
A: 请检查：
1. 是否启用了"使用循环变量"
2. 起始值类型是否选择正确
3. 如果使用全局变量，该变量是否已定义且有值

**Q: 可以在循环次数中使用全局变量吗？**
A: 可以！在"循环次数"输入框中输入 `{{变量名}}`，系统会自动替换为全局变量的值。

**Q: 循环变量和全局变量有什么区别？**
A: 
- 循环变量：每次循环自动递增，只在循环体内有效
- 全局变量：固定值，在整个工作流中有效

**Q: 可以在一个工作流中使用多个循环变量吗？**
A: 可以！每个循环模块可以定义自己的循环变量，变量名可以不同。
